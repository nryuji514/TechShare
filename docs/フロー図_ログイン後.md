# フロー図（ログイン後の画面遷移）

ログインに成功したあと、アプリ内をどう移動するかをまとめた図。
発表ではこの図をスクリーンショットしてスライドに貼る想定。

> すべての画面に `@login_required` がついているため、未ログインの場合はログイン画面に戻される。

![ログイン後フロー図](diagrams/フロー図_ログイン後.png)

<details><summary>Mermaidソース（編集用）</summary>

```mermaid
%%{init: {'theme':'base','themeVariables':{'fontSize':'16px','fontFamily':'Segoe UI, Meiryo, sans-serif','lineColor':'#8a8a8a'},'flowchart':{'curve':'basis','nodeSpacing':50,'rankSpacing':65}}}%%
flowchart TD
    Login([ログイン成功]) --> Home["ホーム<br/>最新の投稿5件"]

    Mypage --> ProfileEdit["プロフィール編集<br/>アイコン変更"]
    ProfileEdit -- "保存" --> Mypage
    Mypage -- "編集" --> Edit
    Mypage -- "削除（確認後）" --> Mypage

    Home -- "ログアウト" --> Logout([ログイン画面へ])
```

## ポイント

- 起点はホーム。ここから「記事一覧」「新規投稿」「マイページ」の3方向に分かれる。
- 「いいね」はどの画面（一覧・詳細）から押しても、押した画面にそのまま戻ってくる。
- 投稿の編集は本人だけが可能（`author=request.user` で制限している）。
- 削除はマイページから。確認をはさんでから消す。
